</ul>

<blockquote>
<p>응답 필터와 요청 필터를 구현하는 메서드, 필터가 적용된 서블릿이 호출될 때마다 수행</p>

<h3 id="필터-체이닝">필터 체이닝</h3>
<blockquote>
<p>클라이언트와 서블릿 사이에 여러 개의 필터 객체들이 정해진 순서에 의해 연결되어 잇는것</p>
</blockquote>
<ol>
<li>xml 2. @webFilter 애너테이션 으로 사용할 수 있음.</li>
</ol>
<h3 id="xml-이용한-필터-등록">XML 이용한 필터 등록</h3>
<pre><code>&lt;filter&gt;
  &lt;filter-name&gt; 필터의 이름
  &lt;filter-class&gt; 필터 클래스 이름
  &lt;init-param&gt; 
    &lt;param-name&gt; 초기화 파라미터 이름
    &lt;param-value&gt; 초기화 파라미터 값</code></pre><blockquote>
<ul>
<li>filter-name : 필터의 이름을 지정하며, 각 필터들은 유일한 값을 가져야 함(생략 불가능</li>
</ul>
</blockquote>
<ul>
<li>filter-class : 패키지를 포함하는 필터 클래스의 이름 (생략 x)</li>
<li>init-param : 초기화 파라미터 지정(생략 o)</li>
<li>param-name : 초기화 파라미터의 이름 지정 (init-param지정시 생략 x)</li>
<li>param-valeu : 초기화 파라미터의 값 지정( // )</li>
</ul>
<pre><code>&lt;filter-mapping&gt;
  &lt;filter-name&gt; 필터의 이름
  &lt;url-pattern&gt; 매핑 이름</code></pre><blockquote>
<ul>
<li>filter-name : 매핑하고자하는 필터의 이름을 지정 (filter-name)</li>
</ul>
</blockquote>
<ul>
<li>url-pattern : 패턴을 사용해 매핑하고자 하는 서블릿 지정(패턴에 만족하는 모든 서블릿 적용)</li>
</ul>
<h3 id="webfilter-매핑">@WebFilter 매핑</h3>
<blockquote>
<p>@WebFilter(filterName=&quot;필터이름&quot; urlPatterns=&quot;URL 패턴&quot;)</p>
</blockquote>
<ul>
<li><p>filterName : 필터 클래스의 필터 이름</p>
</li>
<li><p>urlPatterns : 필터와 매핑이 이루어지는 대상</p>
<blockquote>
<p>ex)
@WebFilter(&quot;/URL 패턴&quot;)
@WebFilter(value=&quot;/first&quot;)
@WebFilter(urlPatterns=&quot;/first&quot;)
@WebFilter(value={&quot;/first&quot;, &quot;/second&quot;})
@WebFilter(urlPatterns={&quot;/first&quot;, &quot;/second&quot;})
@WebFilter(&quot;/*&quot;)
@WebFilter(&quot;*.jsp&quot;) jsp 문서일 경우는 / 를 사용 안함.</p>
</blockquote>
</li>
</ul>
