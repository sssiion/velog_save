<h4 id="요청-정보-처리">요청 정보 처리</h4>
<p>파라미터 관련 메서드</p>
<blockquote>
<p>setCharacterEncoding(encoding) : 클라이언트에서 POST 방식으로 전달된 문자열을 인자로 지정한 문자 집합으로 인코딩
getParameter(String name) : 클라이언트에 전달된 파라미터 값을 추출
getParameterNames(String name) : 클라이언트에서 전달된 파라미터 이름을 추출
getParameterValues(String name) : 클라이언트에서 하나의 파라미터로 전달된 여러개의 값을 추출해 배열로 반환</p>
</blockquote>
<p>HttpServletResponse 객체의 주요 메서드</p>
<blockquote>
<p>getWriter() : 웹서버에서 클라이언트로 데이터 출력을 위한 printWRiter 객체 생성
setContentType(STring type) : 웹서버에서 클라이언트로 전달할 데이터의 타입 MIME 타입을 지정
sendRedirct(url) : url에 지정된 대상으로 리디렉트 수행</p>
</blockquote>
<h4 id="서블릿의-포워딩">서블릿의 포워딩</h4>
<p>sendRedirect(String url) 를 이용해서 포워딩할 때 addHeader(&quot;Refresh&quot;, 지연시간(초);url=Location) 으로 지연 후 실행할 수 있음.</p>
<h5 id="자바스크립트를-이용한-포워딩">자바스크립트를 이용한 포워딩</h5>
<p>location 객체의 href 속성이나 replace() 메서드 사용</p>
<blockquote>
<p>ex)
out.print(&quot;location.href='ftarget';&quot;);
out.print(&quot;location.replace('ftarget');&quot;);</p>
</blockquote>
<h5 id="requestdispatcher-클래스-사용-포워딩">RequestDispatcher 클래스 사용 포워딩</h5>
<p>브라우저가 아닌 서블릿 컨테이너가 포워딩의 주체가 되는 방법</p>
<blockquote>
<p>RequestDispatcher dis = request.getRequestDispatcher(포워딩 대상)
dis.forward(request, response)</p>
</blockquote>
